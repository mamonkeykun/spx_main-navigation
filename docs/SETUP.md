# Setup

## Prerequisites

- Node.js `18.x`
- `nvm` for version pinning; create `.nvmrc` with:

```text
18
```

- npm `9+`
- SPFx `v1.20.0` via `@microsoft/generator-sharepoint`
- Yeoman via `yo`
- `gulp-cli`

Install the global tooling with:

```bash
npm install -g yo gulp-cli @microsoft/generator-sharepoint@1.20.0
```

Recommended VS Code extensions:

- `ms-vscode.vscode-typescript-next`
- `dbaeumer.vscode-eslint`
- `esbenp.prettier-vscode`
- `mdickin.markdown-shortcuts`
- `SharePoint.vscode-sharepoint-extension`

## Initial Setup Steps

1. Clone the repository.

```bash
git clone <repository-url>
cd spx_main-navigation
```

2. Select the pinned Node.js version.

```bash
nvm use
```

3. Install project dependencies.

```bash
npm install
```

4. Copy environment variables and fill in tenant-specific values.

```bash
cp .env.example .env.local
```

Required values:

- `TENANT_URL=https://<tenant>.sharepoint.com`
- `SITE_URL=https://<tenant>.sharepoint.com/sites/<site-name>`

5. Trust the SPFx development certificate.

```bash
gulp trust-dev-cert
```

6. Start the local development workbench host.

```bash
gulp serve
```

7. Open the SharePoint hosted workbench in the tenant site:

```text
https://<tenant>.sharepoint.com/sites/<site-name>/_layouts/15/workbench.aspx
```

For extension testing, load a modern page on the same site with the debug query string generated by `gulp serve`.

## VS Code Workspace Settings

Create `.vscode/settings.json` with the following content:

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "typescript.tsdk": "node_modules/typescript/lib",
  "typescript.enablePromptUseWorkspaceTsdk": true
}
```

## Troubleshooting

### Certificate Trust Issues

macOS:

```bash
gulp untrust-dev-cert
gulp trust-dev-cert
```

If the browser still reports an invalid certificate, open Keychain Access, locate the localhost development certificate created by SPFx, and mark it as trusted.

Windows:

- Run the terminal as Administrator before `gulp trust-dev-cert`.
- If trust fails, remove old localhost developer certificates from the Current User certificate store, then rerun the command.

### Port 4321 Already In Use

Identify the process and stop it, or change the dev server port in `config/serve.json` if the repo standard allows it.

macOS/Linux:

```bash
lsof -i :4321
kill -9 <pid>
```

Windows PowerShell:

```powershell
Get-NetTCPConnection -LocalPort 4321 | Select-Object OwningProcess
Stop-Process -Id <pid> -Force
```

### SharePoint Workbench 403 Error

- Confirm `SITE_URL` points to a site where your account has edit access.
- Verify the tenant allows custom scripting and SPFx package deployment for the target site.
- Make sure you are opening the hosted workbench on the exact tenant and site used for debugging.
- If the site was recently created, wait for provisioning to complete and retry.
